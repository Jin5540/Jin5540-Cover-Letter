1. 지원 동기와 앞으로의 목표

쉽게 만들기보다, 오래 쓰이게 만들고 싶다

사기업 대상 보안 인증 시스템(PKI) 유지보수 업무를 통해 레거시 시스템의 중요성과 지속 가능한 코드의 가치를 깊이 깨달았습니다. 특히 Spring Boot로의 마이그레이션을 주도하며 유지관리성과 성능을 동시에 개선한 경험은 저에게 백엔드 엔지니어로서의 방향성을 명확히 해주었습니다.

Java와 Node.js를 활용한 서버 개발부터 Kotlin 기반 Android 앱 개발까지 폭넓은 기술 경험을 쌓아왔고, 각 프로젝트에서 '안정적인 시스템 구조'를 직접 설계하고 운영하며 실무 역량을 다져왔습니다. 사이드 프로젝트들을 통해서는 기상청 API 연동, 실시간 통신, 블록체인 연동 등 다양한 기술적 도전을 경험하며 지속적으로 성장해왔습니다.

완성도를 더해가는 개발자로서 더욱 견고하고 확장 가능한 시스템을 구축하고, 팀과 함께 기술적 깊이와 비즈니스 가치를 모두 만족시키는 솔루션을 만들어가고 싶습니다.

2. 성격의 장점과 단점

파고드는 문제 해결력과 팀을 하나로 만드는 소통력

PKI 시스템의 Spring Boot 마이그레이션 과정에서 복잡한 레거시 코드를 분석하고 단계별로 개선해나가는 경험을 통해 이를 확인할 수 있었습니다. 또한 주문관리시스템(OMS) 개발에서 API 설계와 UI 개선을 병행하며 업무 효율성을 높인 것처럼, 기술적 구현뿐만 아니라 사용자 관점에서 생각하는 습관을 갖고 있습니다.

지역 모임을 운영하며 온라인과 오프라인에서 지속적으로 커뮤니티를 이끌어가고 있는 경험은 제 소통 능력과 리더십을 보여줍니다. 다양한 참가자들과의 소통을 통해 개발팀에서 필요한 명확한 의사소통과 협업 능력을 자연스럽게 기를 수 있었습니다.

반면 제 단점은 빠른 결과를 추구하다 보니 때로는 안정성보다 속도에 치중하는 경향이 있다는 점입니다. 하지만 이를 극복하기 위해 코드 리뷰와 테스트 주도 개발(TDD)을 적극 도입하고, 단계적 개발을 통해 속도와 품질의 균형을 맞춰가는 연습을 하고 있습니다. 특히 현재 개발 중인 채팅 앱에서는 기능 구현 후 반드시 리팩토링 단계를 거쳐 코드 품질을 높이는 습관을 기르고 있습니다.

3. 직무 관련 경험

개발부터 운영까지

사기업 보안 인증 시스템에서는 Java, Spring Boot, MySQL, Apache 기술 스택으로 시스템 유지보수와 마이그레이션을 담당했습니다. 레거시 시스템을 최신 프레임워크로 전환하며 성능 최적화와 코드 품질 향상을 동시에 달성했습니다.

주문관리시스템 프로젝트에서는 Flask와 JavaScript를 활용해 API 설계부터 UI 개선까지 풀스택 개발을 수행했고, Linux 환경에서의 배포와 운영 경험도 쌓았습니다.

다양한 기술 스택 활용 경험도 보유하고 있습니다. Laravel 기반 스포츠 시설 예약 시스템에서는 ETH 기반 리워드 시스템을 연동하며 블록체인 기술을 실무에 적용했고, Kotlin과 Node.js를 활용한 실시간 커뮤니케이션 앱에서는 WebSocket 기반 실시간 통신을 구현했습니다.

현재 집중하고 있는 프로젝트는 온라인과 오프라인 모임을 연결하는 채팅 애플리케이션입니다. 지역 모임을 운영하며 참가자들 간의 소통 방식을 관찰하던 중, 온라인 채팅과 오프라인 만남을 자연스럽게 연결할 수 있는 플랫폼의 필요성을 느꼈습니다. 이러한 실제 사용자 경험을 바탕으로 기획한 이 프로젝트에서는 Kotlin 기반 Android 개발과 Node.js 서버 개발을 동시에 담당하고 있습니다. 실시간 채팅 기능부터 오프라인 모임 일정 관리까지, 커뮤니티 구성원들이 진정으로 필요로 하는 기능들을 직접 구현하며 사용자 중심의 개발 경험을 쌓고 있습니다.
